## تم تنفيذه الآن (واجهة النموذج)
- تغيير عنوان القسم من **خيارات التفعيل** إلى **احتياجات التنفيذ**.
- تحويل مفاتيح التفعيل إلى Select (نعم/لا) بدل checkbox.
- تحديث المسميات:
  - يوجد راعٍ -> الحاجة لرعاية رسمية
  - يوجد شركاء -> الحاجة لشركاء خارجيين
- إضافة حقول واجهة (UI) لـ:
  - أجندة الحفل
  - المواصلات
  - الصيانة بالموقع
  - الهدايا والدروع
  - مشاركة البرامج
  - الشهادات وكتب الشكر
  - بطاقات الدعوة
- تعديل بند المستلزمات: استبدال (نوع المسؤول/اسم المسؤول) بـ (آلية التأمين + تفاصيل أخرى).
- إزالة قسم **الجهات الشركاء** من النموذج.

## المطلوب التالي (Backend + Data Model)
1. **قاعدة البيانات**
   - إضافة أعمدة/جداول منظمة لكل احتياج تنفيذي جديد.
   - خيار أفضل: جداول child مرتبطة بـ `monthly_activities` بدل أعمدة نصية كثيرة.

2. **التحقق والحفظ**
   - تحديث Validation Rules في `MonthlyActivitiesController@store/update` للحقول الجديدة.
   - ربط الحقول الجديدة بعمليات الحفظ والتحديث ونسخ الإصدارات.

3. **لوحة الأدمن (Lookups)**
   - إنشاء جدول Lookup لخدمات (زها تايم/البرامج) القابلة للتعدد.
   - Seeder ابتدائي بالقيم:
     - حكواتي
     - رسم عالوجوه
     - ساحر
     - تلي ماتش
     - شخصيات
     - أخرى
   - ربطها بشاشة الإدارة الحالية Lookups.

4. **اعتمادات الوحدات مستقبلاً**
   - ربط كل احتياج تنفيذي بالوحدة/المسار الخاص به (علاقات/برامج/نقل/صيانة...) دون تطبيقه الآن.

5. **التقارير والـ API**
   - إظهار الحقول الجديدة في صفحات العرض/التقارير عند التفعيل لاحقاً.
