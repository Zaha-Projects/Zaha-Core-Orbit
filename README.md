# Zaha OPS – Laravel Setup Seed

هذا المستودع مخصص لبدء بناء نظام التشغيل التشغيلي (OPS) لمنصة زها باستخدام Laravel 11 وMySQL. نظراً لقيود الشبكة الحالية (حظر التحميل الخارجي)، تم تجهيز خطة وخطوات bootstrap جاهزة للتنفيذ بمجرد توفر الوصول إلى Packagist/GitHub.

## ما تم إعداده
- ملف `scripts/bootstrap-laravel.sh` لإنشاء مشروع Laravel في جذر المستودع عبر `composer create-project` مع الحفاظ على مجلد `.git`.
- خطة تنفيذ مرحلية تغطي الوحدات الأساسية (الأجندة، الخطة الشهرية، الإيرادات، الصيانة، النقل، التقارير) في `docs/implementation-plan.md`.
- تعليمات تشغيل محلية بعد إتمام bootstrap (تنصيب الاعتمادات، تشغيل السيرفر، وإعداد قاعدة البيانات).

## المتطلبات المسبقة
- PHP 8.2 أو أحدث
- Composer
- MySQL 8
- Node.js 20 (للبناء الأمامي عند الحاجة)

## خطوات تشغيل bootstrap عند توفر الشبكة
1. تأكد من أن جذر المستودع لا يحتوي ملفات Laravel (لا يوجد ملف `artisan`).
2. نفّذ:
   ```bash
   ./scripts/bootstrap-laravel.sh
   ```
   سيقوم السكربت بإنشاء مشروع Laravel في مجلد مؤقت ثم ينسخ الملفات إلى الجذر مع ترك `.git` كما هو.

> **ملاحظة:** داخل بيئة التطوير الحالية فشل تحميل الحزم بسبب خطأ proxy (403). حالما يتم فتح الاتصال، أعد تشغيل السكربت أعلاه.

## بعد نجاح bootstrap
1. انسخ ملف البيئة:
   ```bash
   cp .env.example .env
   php artisan key:generate
   ```
2. حدّث إعدادات قاعدة البيانات في `.env`.
3. ثبّت الحزم الأمامية (اختياري للـ Blade/Vite):
   ```bash
   npm install
   npm run build
   ```
4. شغّل الخادم التطويري:
   ```bash
   php artisan serve
   ```

## هيكلة العمل القادمة
- التفاصيل الكاملة للخطة الزمنية والدفعات موجودة في `docs/implementation-plan.md`.
- يمكن إضافة ملفات تصميم قاعدة البيانات (مثل ERD/DBML من وثيقة SRS) داخل `docs/` لاحقاً لربطها بالهجرة (migrations).
